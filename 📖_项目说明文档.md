# Quartz 项目结构详解

> 本文档详细介绍 Quartz 项目的目录结构和各个文件的作用。

---

## 📁 项目根目录结构

```
worknotes/                          # 你的 Quartz 项目根目录
├── content/                        # 📝 你的笔记文件夹（最重要！）
│   ├── index.md                    # 首页内容
│   ├── Obsidian 入门指南.md        # 示例笔记
│   ├── Quartz 使用教程.md          # 示例笔记
│   └── 知识管理方法论.md           # 示例笔记
│
├── quartz.config.ts                # ⚙️ 主配置文件（网站行为）
├── quartz.layout.ts                # 🎨 布局配置文件（页面结构）
│
├── public/                        # 📦 构建输出目录（自动生成）
│   ├── index.html                 # 首页
│   ├── static/                    # 静态资源
│   └── ...                        # 其他生成的页面
│
├── docs/                          # 📚 Quartz 官方文档
│   ├── configuration.md           # 配置文档
│   ├── layout.md                  # 布局文档
│   ├── hosting.md                 # 部署文档
│   └── features/                  # 功能特性文档
│
├── quartz/                        # 🔧 Quartz 核心代码（一般不需修改）
│   ├── components/                # UI 组件（按钮、卡片等）
│   ├── plugins/                   # 插件（处理 Markdown）
│   ├── styles/                    # 样式文件
│   └── util/                     # 工具函数
│
├── .github/                       # 🤖 GitHub 配置
│   └── workflows/                 # GitHub Actions 工作流
│       └── forcesdetach.yml       # 自动化部署脚本
│
├── package.json                   # 📋 Node.js 项目配置
├── tsconfig.json                  # TypeScript 配置
├── .gitignore                     # Git 忽略文件
└── .npmrc                         # npm 配置
```

## 📝 关键文件详解

### 1️⃣ content/ - 你的笔记文件夹

这是**最重要的文件夹**，你所有的笔记都放在这里。

```
content/
├── index.md                    # 首页内容
├── 笔记1.md                   # 普通笔记
├── 笔记2.md
└── 子文件夹/                   # 可以创建子文件夹组织笔记
    ├── 子笔记1.md
    └── 子笔记2.md
```

**笔记的 Frontmatter（前置元数据）示例：**

```yaml
---
# 标题 - 显示在页面和搜索结果中
title: 我的笔记标题

# 标签 - 用于分类和组织笔记
tags:
  - 学习
  - 笔记
  - 重要

# 创建日期
created: 2024-01-15

# 修改日期（可选，不填则自动使用 Git 记录的时间）
modified: 2024-01-20

# 是否为草稿（草稿不会发布）
draft: false

# 自定义 slug（可选，影响 URL）
slug: my-custom-slug

# 自定义别名（可以通过别名访问）
aliases:
  - 旧笔记名
  - 笔记简称
---
```

---

### 2️⃣ quartz.config.ts - 主配置文件

**文件位置：** 项目根目录

**作用：** 控制网站的基础行为和外观

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `pageTitle` | 网站标题 | `"我的数字花园"` |
| `pageTitleSuffix` | 标题后缀 | `"🌿"` |
| `enableSPA` | 单页应用模式 | `true` |
| `enablePopovers` | 悬停预览 | `true` |
| `baseUrl` | 网站部署 URL | `"username.github.io"` |
| `locale` | 语言/地区 | `"zh-CN"` |
| `defaultDateType` | 日期类型 | `"modified"` |

**详细配置说明请查看文件内的中文注释！**

---

### 3️⃣ quartz.layout.ts - 布局配置文件

**文件位置：** 项目根目录

**作用：** 控制页面的布局结构和组件

页面布局区域：
```
┌─────────────────────────────────────┐
│            header (页眉)             │
├──────┬──────────────────────┬──────┤
│      │      beforeBody       │      │
│      ├──────────────────────┤      │
│ left │                      │ right │
│      │     center (内容)     │      │
│      │                      │      │
│      ├──────────────────────┤      │
│      │      footer (页脚)    │      │
└──────┴──────────────────────┴──────┘
```

**详细配置说明请查看文件内的中文注释！**

---

### 4️⃣ quartz/styles/ - 样式文件夹

```
quartz/styles/
├── base.scss         # 基础样式（一般不修改）
├── variables.scss    # 变量定义（颜色、断点等）
├── callouts.scss     # 提示块样式
├── syntax.scss       # 代码高亮样式
└── custom.scss       # 🎨 你的自定义样式（这里添加美化和调整！）
```

**custom.scss 是你添加自定义样式的地方！**

---

### 5️⃣ package.json - 项目配置

```json
{
  "name": "@jackyzha0/quartz",      // 项目名称
  "version": "4.5.2",               // 版本号
  "scripts": {
    "quartz": "./quartz/bootstrap-cli.mjs",  // Quartz CLI
    "docs": "npx quartz build --serve -d docs",  // 本地预览文档
    "check": "tsc --noEmit && npx prettier . --check",  // 类型检查
    "format": "npx prettier . --write"  // 格式化代码
  },
  "engines": {
    "node": ">=22",                 // Node.js 版本要求
    "npm": ">=10.9.2"               // npm 版本要求
  }
}
```

---

### 6️⃣ .github/workflows/ - GitHub Actions

```
.github/workflows/
└── forcesdetach.yml       # 自动化部署工作流
```

当你 push 代码到 GitHub 时，这个工作流会自动：
1. 安装依赖
2. 构建网站
3. 部署到 GitHub Pages

## 🎯 常用命令

| 命令 | 说明 |
|------|------|
| `npx quartz build` | 构建网站（输出到 public/） |
| `npx quartz build --serve` | 构建并本地预览 |
| `npx quartz create` | 初始化新项目 |
| `npx quartz check` | 检查配置是否正确 |

## 📌 快速参考

### 常见任务

**Q: 如何修改网站标题？**
A: 编辑 `quartz.config.ts` 中的 `pageTitle`

**Q: 如何添加新笔记？**
A: 在 `content/` 文件夹中创建新的 `.md` 文件

**Q: 如何修改主题颜色？**
A: 编辑 `quartz.config.ts` 中的 `theme.colors`

**Q: 如何添加自定义样式？**
A: 编辑 `quartz/styles/custom.scss`

**Q: 如何禁用某个功能？**
A: 在 `quartz.config.ts` 中注释掉对应的插件
